= Deployment Guide
:toc:

== Voraussetzungen

AWS-Profil `praxis-wabner` muss konfiguriert sein:

[source,bash]
----
aws configure --profile praxis-wabner
# AWS Access Key ID: (siehe KeyPass "deploy-praxis-wabner")
# AWS Secret Access Key: (siehe KeyPass)
# Region: eu-west-1
----

== Deployment-Befehl

[source,bash]
----
mvn clean package -Paws && mvn exec:exec@s3-sync -Paws && mvn exec:exec@cloudfront-invalidate-de -Paws && mvn exec:exec@cloudfront-invalidate-com -Paws
----

Das macht:

1. `package` - Kopiert Website-Dateien nach `dist/`
2. `s3-sync` - Synchronisiert `dist/` nach S3-Bucket `praxis-wabner.de`
3. `cloudfront-invalidate-de` - Invalidiert CloudFront-Cache für .de Domain
4. `cloudfront-invalidate-com` - Invalidiert CloudFront-Cache für .com Domain

== CSS Cache-Busting

IMPORTANT: Bei CSS-Änderungen den localStorage-Cache-Key hochzählen!

=== Warum?

Die Website cached CSS im localStorage des Browsers. Ohne Key-Änderung sehen Besucher alte Styles.

=== Wie?

In *allen* HTML-Dateien im `<head>`-Bereich den Key ändern:

[source,javascript]
----
var b="sourceMain2";  // → sourceMain3, sourceMain4, etc.
----

Aktueller Stand: `sourceMain2`

=== Schnellbefehl

[source,bash]
----
# Alle HTML-Dateien auf einmal ändern (z.B. von sourceMain2 auf sourceMain3):
sed -i '' 's/sourceMain2/sourceMain3/g' *.html
----
