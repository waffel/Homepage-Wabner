== Migration Steps

=== Phase 1: Preparation

* [ ] List all WorkMail users and aliases
* [ ] List all domains in Route 53 used for email
* [ ] Export current mailboxes from WorkMail
* [ ] Create Mailbox.org account

=== Phase 2: Mailbox.org Setup

* [ ] Create users in Mailbox.org
* [ ] Add custom domains
* [ ] Configure aliases to match current setup
* [ ] Verify domain ownership (DNS TXT record)

=== Phase 3: Test Migration (1 User)

* [ ] Import mailbox data for test user
* [ ] Configure Thunderbird with new IMAP settings
* [ ] Configure Android with new settings
* [ ] Configure Mac Mail with new settings
* [ ] Test send/receive
* [ ] Test calendar sync (if used)

=== Phase 4: DNS Cutover

* [ ] Update MX records in Route 53
* [ ] Update SPF record
* [ ] Add DKIM record
* [ ] Remove old WorkMail DNS records
* [ ] Wait for propagation (up to 48h)

=== Phase 5: Full Migration

* [ ] Migrate remaining users
* [ ] Verify all clients working
* [ ] Monitor for issues

=== Phase 6: Cleanup

* [ ] Delete WorkMail organization
* [ ] Update documentation
* [ ] Close this migration

== DNS Changes Required

=== Current (WorkMail) - TO BE REMOVED

[source,dns]
----
MX    @    10  inbound-smtp.eu-central-1.amazonaws.com
TXT   @    "v=spf1 include:amazonses.com ~all"
CNAME autodiscover  autodiscover.mail.eu-central-1.awsapps.com
----

=== New (Mailbox.org) - TO BE ADDED

[source,dns]
----
MX    @    10  mx1.mailbox.org
MX    @    20  mx2.mailbox.org
MX    @    30  mx3.mailbox.org

TXT   @    "v=spf1 include:mailbox.org ~all"

CNAME mbo0001._domainkey  mbo0001._domainkey.mailbox.org

SRV   _autodiscover._tcp  0 0 443 mailbox.org
----

== Rollback Plan

If issues occur after DNS cutover:

. Revert MX records to WorkMail
. Wait for propagation
. Investigate issues
. Retry migration
